#### 2. **Swarm Synthesis Reverb** (Soundtoys SpaceBlender Feature)
**What It Does**:
- Creates dense, textured reverb spaces using particle/swarm algorithms
- Each particle is a delayed/filtered copy with slight variations
- Swarm behavior (attraction, repulsion, randomness) creates unique textures
- NOT algorithmic (Freeverb) or convolution (impulse response)

**Why It's Useful**:
- Unique reverb character unavailable elsewhere
- Dense, "living" reverb tails
- Ideal for ambient, cinematic, experimental music

**Implementation Difficulty**: ‚≠ê‚≠ê‚≠ê‚≠ê (High)
**Files Needed**: `Sources/DSP/SwarmReverb.cpp/h` (~600 lines)

**Algorithm**:
```cpp
// 1. Create N particles (100-1000) in 3D space
// 2. Each particle = delay line + filter + gain
// 3. Apply swarm rules:
//    - Attraction to center (cohesion)
//    - Repulsion from neighbors (separation)
//    - Random movement (chaos)
// 4. Update particle positions each sample
// 5. Mix all particle outputs with distance-based gain
```

#### 3. **Humanizer/Movement Generator** (Naturaliser 2 Feature)
**What It Does**:
- Adds subtle, time-sliced variations to audio
- 4 dimensions: Spectral, Transient, Colour (tone), Noise
- Slices audio into musical steps (16th notes, 8th notes, etc.)
- Each slice gets slightly different processing
- Creates "imperfect" human-like feel

**Why It's Useful**:
- Makes programmed drums/MIDI sound human
- Adds life to static loops
- Subtle movement for ambient textures

**Implementation Difficulty**: ‚≠ê‚≠ê‚≠ê‚≠ê (High)
**Files Needed**: `Sources/DSP/AudioHumanizer.cpp/h` (~500 lines)

**Algorithm**:
```cpp
// 1. Slice audio into musical time divisions (sync to BPM)
// 2. For each slice, apply random variations:
//    - Spectral: ¬±0.5dB per frequency band (50-100 bands)
//    - Transient: ¬±10% attack/sustain
//    - Colour: ¬±2% filter cutoff/resonance
//    - Noise: ¬±3dB noise floor

```
