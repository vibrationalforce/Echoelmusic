Security

Updated: February 2026

Privacy-First Design
Your data stays on your device. Echoelmusic uses a local-first architecture with platform-native encryption and authentication. No servers, no accounts, no tracking.

Local-First Architecture

Echoelmusic processes all data locally on your device. We do not operate external servers or cloud backends for user data. This means:

  • No user accounts: No registration, no passwords to leak
  • No data collection: We do not collect, store, or transmit your personal data
  • No analytics tracking: No third-party analytics or ad SDKs
  • No external dependencies: Built entirely on Apple native frameworks

Encryption

  • Data at Rest: Local data protected using Apple Data Protection
  • AES-256: Strong encryption for sensitive local storage via Apple CryptoKit
  • TLS 1.2/1.3: All network connections (collaboration, streaming) use encrypted transport
  • Certificate Pinning: Protection against man-in-the-middle attacks on network requests

Authentication

  • Face ID / Touch ID: Optional biometric authentication for sensitive features (Apple devices)
  • Optic ID: Supported on Apple Vision Pro
  • Keychain / Keystore: Credentials stored in platform-native secure enclaves
  • Device Integrity: Basic integrity checks to protect against tampered environments

Biometric Data Handling

Heart rate, HRV, and other biometric data read from Apple HealthKit is:

  • Processed locally: All biometric-to-audio mapping happens on-device in real time
  • Never transmitted: Biometric data is not sent to any server or third party
  • Not stored permanently: Session data exists only during active use
  • User-controlled: Biometric access requires explicit permission and can be revoked at any time

Echoelmusic is not a medical device. Biometric readings are used for creative and entertainment purposes only. See our Health Disclaimer.

Data Protection Compliance

  • GDPR: Compliant with EU General Data Protection Regulation — minimal data collection by design
  • CCPA: Compliant with California Consumer Privacy Act — no personal information sold or shared
  • App Store Guidelines: Reviewed against Apple App Store and Google Play security requirements
  • Privacy by Design: Data minimization and local processing are core architectural principles, not afterthoughts

For full details on data handling, see our Privacy Policy.

Open Source & Transparency

Echoelmusic uses only native Apple frameworks (Core Audio, HealthKit, Metal, AVFoundation, CryptoKit). No third-party SDKs, analytics, or ad networks are included in the app.

We believe transparency builds trust. If you have questions about our security practices, we are happy to discuss them.

Limitations

No software is perfectly secure. While we follow security best practices, we cannot guarantee absolute security. Specifically:

  • Security depends on your device's OS version and security updates
  • Collaboration and streaming features involve network communication governed by third-party platform policies
  • Biometric sensor accuracy depends on hardware and conditions outside our control
  • We encourage you to keep your device updated and use strong device passcodes
